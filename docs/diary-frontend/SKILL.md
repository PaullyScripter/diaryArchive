# DiaryArchive Frontend Skill

## Mission
Build a calm, writing-first experience inspired by Archive of Our Own and early blogging platforms.

## Stack
- Next.js 15
- React 19
- TypeScript
- Tailwind CSS
- shadcn/ui
- TanStack Query
- Zustand
- Tiptap

## Design Rules
- Writing always comes first.
- Responsive and accessible (WCAG AA).
- Keyboard-first navigation.
- Dark and light themes.
- Minimal animations.
- Readability over visual effects.

## Components
Create reusable components.
Prefer composition over inheritance.
Keep business logic outside UI components.

## Pages
- Home
- Explore
- Diary Reader
- Diary Editor
- Profile
- Settings
- Notifications
- Authentication
- Admin Dashboard

## Homepage
Include:
- Latest Public Diaries
- Random Diary
- Recently Updated
- Browse by Tags
- Browse by Emotions
- Browse by Year
- Browse by Month

Avoid infinite scrolling and engagement-driven feeds.

## Editor
Use Tiptap with autosave, drafts, markdown shortcuts, drag-and-drop uploads, rich media, undo/redo, and keyboard shortcuts.

Always explain major UI decisions before implementing them.
